# 高并发高流量场景 - 待做事项

## ★★★ 很重要

| 项 | 说明 |
|----|------|
| 订单幂等 | `idempotencyKey` + Redis SetNX，防重复下单，外卖场景必问 |
| 缓存 TTL 随机抖动 | `ttl + rand(600s)`，避免雪崩 |
| 令牌桶限流 | `golang.org/x/time/rate` 替代滑动窗口，更平滑 |
| Redlock | 多 Redis 节点分布式锁，提高可用性 |

## ★★ 重要

| 项 | 说明 |
|----|------|
| 本地缓存 | 热点偏好/商家加进程内 LRU，减少 Redis 访问 |
| 布隆过滤器 | 防穿透，拦截不存在的 userID/merchantID |
| 热点 key 互斥锁 | 击穿时只放一个请求回源 |
| 索引优化 | `(user_id, created_at)` 等联合索引 |

## ★ 加分项

| 项 | 说明 |
|----|------|
| Kafka 异步写 | 偏好/订单写库异步化，降低 RT |
| 分级限流 | VIP 更高配额、爬虫 IP 封禁 |
| 连接池动态调参 | 结合 Prometheus 调整 MaxOpenConns |

## 面试优先做的 3 个

1. **幂等性**：外卖下单必问，当前缺失
2. **缓存雪崩的 TTL 抖动**：改动小，能体现生产经验
3. **布隆过滤器**：能讲清楚原理和适用场景即可
